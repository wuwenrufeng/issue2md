# 任务跟踪指令

**重要**: 当你在执行 `tasks.md` 中的任务时，必须遵守以下规则：

## 📝 任务完成后的必做事项

### 1. 标记已完成任务
每当完成 tasks.md 中的一个任务后，**必须**在该任务编号前添加 `✅` 标记：

**示例：**
```markdown
- **任务 2.1.1** ✅ 编写测试文件
- **任务 2.1.2** ❌ 编写测试用例  # 未完成
```

### 2. 更新进度概览表
完成任务后，**必须**更新文档顶部的进度概览表：
- 重新计算完成数量
- 更新百分比
- 更新状态（✅ 完成 / ⚠️ 部分完成 / ❌ 未开始）

### 3. 自动触发时机
以下情况**必须**更新 tasks.md：
- ✅ 完成一个任务编号（如 2.1.1）
- ✅ 所有测试通过
- ✅ 代码实现完成并提交
- ✅ 重构完成

## 🎯 执行流程

1. **开始任务前**：查看 tasks.md，确认当前要做的任务编号
2. **执行任务**：按照任务要求实现功能
3. **验证完成**：运行测试，确保功能正常
4. **更新标记**：编辑 `specs/001-core-functionality/tasks.md`，添加 ✅ 标记
5. **更新进度**：更新顶部的进度概览表

## 📌 示例

### 场景：完成 Phase 2.1.1

```bash
# 1. 开始任务
- **任务 2.1.1** 编写测试文件

# 2. 完成测试文件编写

# 3. 更新标记
- **任务 2.1.1** ✅ 编写测试文件

# 4. 更新进度表（如果 2.1.x 全部完成）
| Phase 2 | URL Parser（URL解析器） | ⚠️ 进行中 | 5/10 (50%) |
```

## ⚠️ 注意事项

- **不要跳过标记**：即使你认为自己记住了，也要写下来
- **及时更新**：任务完成后立即更新，不要累积
- **保持同步**：tasks.md 是唯一的进度来源，必须与实际代码同步

---

**记住**: tasks.md 是你的开发地图，保持它的准确性对后续工作至关重要！
